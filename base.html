Contour

is a different take on CSS libraries

Foreword

Contour was conceived to be an alternative to both utility-first and component based libraries. It aims to provide flexibility to the UI implementors while maintaining a clear separation between style and content.

Instead of providing the a set of opinionated implementation or atomic classes, Contour provide a flexible foundation that allows rich customizability and a list of reference variantions based on this foundation. It then makes this foundation scalable by using CSS layers to manage specificity and CSS custom properties to reinforce consistency.

Convention

On the surface, the naming of classes follows the BEM convetion. In Contour, however, the convetion is given new meaning:

component__element--variation

Button

The button selector decorates either <a> or <button>. Use the former if you want it to lead to another page. Use the latter if you want it to perform certain actions.
